# fineReport-builder V2 数据模型设计

## 1. 数据库数据源配置

```json
{
  "name": "sales_data",
  "type": "database",
  "database": "cfs-report",
  "sql": "SELECT region, product, amount, quantity FROM sales WHERE date >= '${startDate}' AND date <= '${endDate}'",
  "column_mapping": {
    "A": "region",
    "B": "product",
    "C": "amount",
    "D": "quantity"
  }
}
```

**字段说明：**
- `name`: 数据源名称
- `type`: 固定为 "database"
- `database`: 数据库连接名称
- `sql`: SQL 查询语句，支持 `${参数名}` 占位符
- `column_mapping`: Excel 列到数据库字段的映射（不支持一对多）

## 2. Class 数据源配置

```json
{
  "name": "credit_data",
  "type": "class",
  "class_name": "com.yocyl.fr.engine.tableData.finance.CreditContractDetailData",
  "return_fields": [
    {"name": "contractNo", "type": "string", "description": "合同编号"},
    {"name": "amount", "type": "number", "description": "金额"},
    {"name": "orgName", "type": "string", "description": "组织名称"}
  ],
  "parameter_template": {
    "orgId": "",
    "startDate": "2026-03-02",
    "endDate": "",
    "indexInfo": [],
    "dimensions": []
  }
}
```

**字段说明：**
- `name`: 数据源名称
- `type`: 固定为 "class"
- `class_name`: Java 类全路径
- `return_fields`: 返回字段列表
- `parameter_template`: 入参模板
  - 空值 `""` → 从筛选组件获取
  - 有值 `"2026-03-02"` → 写死的默认值
  - 空数组 `[]` → 从筛选组件获取（数组类型）

## 3. 筛选组件配置

### 3.1 组件布局

**系统自动生成 Label + 输入控件配对：**

```
┌─────────────────────────────────────────────────────────────────────┐
│  参数面板布局（一行 5 对组件）                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  [开始日期] [日期控件]  [结束日期] [日期控件]  [区域] [下拉框]       │
│      ↑        ↑          ↑        ↑          ↑       ↑              │
│    Label   输入控件    Label   输入控件    Label  输入控件          │
│                                                                     │
│  自动生成独立的 Label 控件（com.fr.form.ui.Label）                   │
│  Label 显示 label 文本，输入控件绑定 code 参数                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 JSON 配置

```json
[
  {
    "label": "开始日期",
    "code": "startDate",
    "type": "DateEditor",
    "default_value": ""
  },
  {
    "label": "结束日期",
    "code": "endDate",
    "type": "DateEditor",
    "default_value": ""
  },
  {
    "label": "组织机构",
    "code": "orgId",
    "type": "TreeComboBoxEditor",
    "default_value": ""
  },
  {
    "label": "合同编号",
    "code": "contractNo",
    "type": "TextEditor",
    "default_value": ""
  },
  {
    "label": "区域",
    "code": "region",
    "type": "ComboBox",
    "default_value": "",
    "options": {
      "east": "华东",
      "south": "华南",
      "north": "华北",
      "west": "西部"
    }
  }
]
```

### 3.3 字段说明

| 字段 | 说明 | 必填 |
|------|------|------|
| `label` | **Label 控件文本**，用户看到的标签 | ✅ |
| `code` | **参数名**，用于传值给数据集 | ✅ |
| `type` | 控件类型 | ✅ |
| `default_value` | 默认值 | ❌ |
| `placeholder` | 占位提示文字 | ❌ |
| `options` | 下拉选项（仅 ComboBox 类型） | ❌ |

### 3.4 控件类型

| 类型 | 说明 | .cpt class |
|------|------|------------|
| TextEditor | 文本输入框 | com.fr.form.ui.TextEditor |
| DateEditor | 日期选择器 | com.fr.form.ui.DateEditor |
| ComboBox | 下拉选择框 | com.fr.form.ui.ComboBox |
| TreeComboBoxEditor | 树形下拉 | com.fr.form.ui.TreeComboBoxEditor |
| ComboCheckBox | 多选下拉 | com.fr.form.ui.ComboCheckBox |
| NumberEditor | 数字输入框 | com.fr.form.ui.NumberEditor |

### 3.5 生成的 .cpt 结构

```xml
<!-- 参数面板 -->
<ReportParameterAttr>
  <ParameterUI class="com.fr.form.main.parameter.FormParameterUI">
    <Layout class="com.fr.form.ui.container.WParameterLayout">
      <!-- 组件对 1: Label + DateEditor -->
      <Widget class="...BoundsWidget">
        <InnerWidget class="com.fr.form.ui.Label">
          <WidgetName name="label_0"/>
          <widgetValue><O>开始日期</O></widgetValue>
        </InnerWidget>
        <BoundsAttr x="10" y="10" width="70" height="28"/>
      </Widget>
      <Widget class="...BoundsWidget">
        <InnerWidget class="com.fr.form.ui.DateEditor">
          <WidgetName name="startDate"/>
        </InnerWidget>
        <BoundsAttr x="90" y="10" width="135" height="28"/>
      </Widget>
      <!-- 更多组件对... -->
    </Layout>
  </ParameterUI>
</ReportParameterAttr>
```

## 4. 样式配置

### 4.1 默认样式集

系统内置 5 种默认样式（参考 FinanceCreditContractAnalysis.cpt）：

| 索引 | 名称 | 说明 | 对齐 | 背景 | 字体颜色 |
|------|------|------|------|------|----------|
| 0 | 表头左列 | 左侧固定列表头 | 左 | 蓝色 | 黑色 |
| 1 | 表头 | 普通表头 | 左 | 蓝色 | 黑色 |
| 2 | 数据 | 普通数据单元格 | 左 | 无 | 黑色 |
| 3 | 金额 | 金额/数值单元格 | 右 | 白色 | 蓝色 |
| 4 | 默认 | 默认样式 | 中 | 无 | 黑色 |

### 4.2 自定义样式

```json
[
  {
    "name": "表头样式",
    "horizontal_alignment": "2",
    "font": {"name": "SimSun", "style": "0", "size": "80"},
    "background": "-1447425",
    "border": true
  },
  {
    "name": "金额样式",
    "horizontal_alignment": "4",
    "font": {"name": "SimSun", "style": "0", "size": "80", "color": "-8163329"},
    "background": "-1",
    "border": true,
    "format": "#,##0.00"
  }
]
```

### 4.3 样式字段说明

| 字段 | 说明 | 值 |
|------|------|-----|
| `horizontal_alignment` | 水平对齐 | 0=中, 2=左, 4=右 |
| `font.name` | 字体名称 | SimSun, 宋体, simhei |
| `font.style` | 字体样式 | 0=常规, 1=粗体, 2=斜体 |
| `font.size` | 字号 | 80=10pt (单位: 1/10pt) |
| `font.color` | 字体颜色 | 颜色值，如 -8163329 |
| `background` | 背景色 | null=透明, -1=白色, -1447425=蓝色 |
| `border` | 是否有边框 | true/false |
| `format` | 数字格式 | "#,##0.00", "#,##0" |

### 4.4 颜色对照

| 颜色 | 值 | 说明 |
|------|-----|------|
| 白色 | -1 | 纯白 |
| 蓝色背景 | -1447425 | 淡蓝色（#E6F7FF） |
| 蓝色字体 | -8163329 | 深蓝色 |
| 边框灰 | -2432266 | 淡灰色边框 |

## 6. 参数绑定流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    参数绑定流程                                  │
└─────────────────────────────────────────────────────────────────┘

1. 用户配置筛选组件
   ┌─────────────────────────────────┐
   │ 开始日期: startDate             │
   │ 组织机构: orgId                 │
   │ 合同编号: contractNo            │
   └─────────────────────────────────┘
              │
              ▼
2. 系统自动检测数据源入参
   ┌─────────────────────────────────┐
   │ 数据库数据集 SQL:               │
   │   ...WHERE date >= '${startDate}'│
   │                                 │
   │ Class 数据集入参:               │
   │   {"orgId": "", "startDate": ""}│
   └─────────────────────────────────┘
              │
              ▼
3. 自动匹配绑定
   ┌─────────────────────────────────┐
   │ startDate → 筛选组件 startDate  │
   │ orgId → 筛选组件 orgId          │
   │                                 │
   │ ⚠️ 警告: indexInfo 无对应组件   │
   └─────────────────────────────────┘
              │
              ▼
4. 生成 .cpt 文件
   - 参数面板：Label + 输入控件配对
   - 数据源参数自动绑定
   - 样式自动应用
```

## 7. 单元格配置

### 7.1 静态文本单元格

```json
{
  "column": 0,
  "row": 0,
  "value": "区域",
  "style_index": 0
}
```

### 7.2 数据列单元格

```json
{
  "column": 0,
  "row": 1,
  "value_type": "DSColumn",
  "data_source": "sales_data",
  "column_name": "region",
  "expand_dir": 0,
  "style_index": 2
}
```

### 7.3 字段说明

| 字段 | 说明 |
|------|------|
| `column` | 列索引（0 开始） |
| `row` | 行索引（0 开始） |
| `value` | 静态值 |
| `value_type` | 值类型：text/DSColumn/Formula |
| `data_source` | 数据源名称 |
| `column_name` | 字段名 |
| `expand_dir` | 扩展方向：0=纵向, 1=横向 |
| `style_index` | 样式索引 |
