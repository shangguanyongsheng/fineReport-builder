## 数据源配置

### 1. 数据库数据源

```json
{
  "name": "sales_data",
  "type": "database",
  "database": "cfs-report",
  "sql": "SELECT region, product, amount, quantity FROM sales WHERE date >= '${startDate}' AND date <= '${endDate}' AND org_id = '${orgId}'"
}
```

**说明：**
- `name`: 数据源名称（自定义）
- `type`: 固定为 `database`
- `database`: 数据库连接名称
- `sql`: SQL 查询语句，使用 `${参数名}` 作为占位符

**SQL 参数示例：**
```sql
-- 参数会从筛选组件获取值
SELECT * FROM sales 
WHERE date >= '${startDate}' 
  AND date <= '${endDate}'
  AND org_id = '${orgId}'
```

---

### 2. Class 数据源

```json
{
  "name": "credit_data",
  "type": "class",
  "class_name": "com.yocyl.fr.engine.tableData.finance.CreditContractDetailData",
  "return_fields": [
    {"name": "contractNo", "type": "string"},
    {"name": "amount", "type": "number"}
  ],
  "parameter_template": {
    "orgId": "",
    "startDate": "2026-03-01",
    "endDate": "",
    "indexInfo": []
  }
}
```

**说明：**
- `name`: 数据源名称（自定义）
- `type`: 固定为 `class`
- `class_name`: Java 类全路径
- `return_fields`: 返回字段列表
  - `name`: 字段名
  - `type`: 字段类型（string/number/date）
- `parameter_template`: 入参模板
  - **空值 `""`** → 从筛选组件获取
  - **有值 `"2026-03-01"`** → 写死的默认值
  - **空数组 `[]`** → 从筛选组件获取（数组类型）

**入参模板规则：**
```json
{
  "orgId": "",               // 空 → 需要配置筛选组件 orgId
  "startDate": "2026-03-01", // 有值 → 固定默认值
  "indexInfo": []            // 空数组 → 需要配置筛选组件 indexInfo
}
```

### JSON 配置

```json
[
  {
    "label": "开始日期",
    "code": "startDate",
    "type": "DateEditor",
    "default_value": ""
  },
  {
    "label": "组织机构",
    "code": "orgId",
    "type": "TreeComboBoxEditor",
    "default_value": ""
  },
  {
    "label": "合同编号",
    "code": "contractNo",
    "type": "TextEditor",
    "default_value": ""
  },
  {
    "label": "区域",
    "code": "region",
    "type": "ComboBox",
    "default_value": "",
    "options": {
      "east": "华东",
      "south": "华南",
      "north": "华北"
    }
  }
]
```

### 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| `label` | **Label 文本**，自动生成独立 Label 控件 | "开始日期" |
| `code` | **参数名**，绑定到输入控件 | "startDate" |
| `type` | 控件类型 | DateEditor |
| `default_value` | 默认值 | "" |
| `options` | 下拉选项（仅 ComboBox） | {"key": "显示值"} |

### 支持的控件类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| `TextEditor` | 文本输入框 | 合同号、名称等 |
| `DateEditor` | 日期选择器 | 开始日期、结束日期 |
| `ComboBox` | 下拉选择框 | 区域、状态等固定选项 |
| `TreeComboBoxEditor` | 树形下拉 | 组织机构、部门等层级结构 |
| `NumberEditor` | 数字输入框 | 金额、数量等 |
| `ComboCheckBox` | 多选下拉 | 多个选项 |

## 样式配置

### 默认样式集

如果不配置 `styles`，系统使用 5 种默认样式：

```json
// styles: [] 时自动使用以下样式
[
  {"name": "表头左列", "horizontal_alignment": "2", "background": "-1447425", "border": true},
  {"name": "表头", "horizontal_alignment": "2", "background": "-1447425", "border": true},
  {"name": "数据", "horizontal_alignment": "2", "border": true},
  {"name": "金额", "horizontal_alignment": "4", "format": "#,##0.00", "border": true},
  {"name": "默认", "horizontal_alignment": "0"}
]
```

### 样式索引对照

| 索引 | 名称 | 用途 |
|------|------|------|
| 0 | 表头左列 | 第一列表头（固定列） |
| 1 | 表头 | 普通表头 |
| 2 | 数据 | 普通数据单元格 |
| 3 | 金额 | 数值/金额单元格（右对齐+千分位） |
| 4 | 默认 | 默认样式 |

```
              ▼                  ▼               ▼
```

## 常见问题

### Q1: JSON 格式错误

**错误**: `Expected ':' after property name in JSON`

**解决**: 
- 属性名用双引号
- 不要用单引号
- 最后一个元素后不要加逗号

**正确**:
```json
{"name": "field1", "type": "string"}
```

**错误**:
```json
{name: "field1", 'type': 'string'}  // ❌
{"name": "field1",}  // ❌
```

### Q2: 参数绑定警告

**现象**: 提示"xxx 参数无对应筛选组件"

**解决**:
1. 检查数据源入参中的参数名
2. 添加对应的筛选组件，确保 `code` 与参数名一致
3. 或者在入参模板中设置默认值

### Q3: 样式不生效

**检查**:
1. `style_index` 是否在有效范围内（0-4）
2. 金额字段使用 `style_index: 3`（右对齐+千分位）
3. 自定义样式需完整配置

## 快速复制模板

### 数据库数据源模板
```json
{
  "name": "数据源名称",
  "type": "database",
  "database": "数据库名",
  "sql": "SELECT 字段1, 字段2 FROM 表名 WHERE 条件 = '${参数名}'"
}
```

```json
{
  "name": "数据源名称",
  "type": "class",
  "class_name": "com.xxx.XXXData",
  "return_fields": [
    {"name": "字段1"},

  ],
  "parameter_template": {
    "参数1": "",
    "参数2": "默认值"
  }
```

```json
[
```
