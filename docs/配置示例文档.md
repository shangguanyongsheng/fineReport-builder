```json
{

    "parameter_template": {

      "endDate": "",

    "A": "contractNo",
```

## 数据源配置

### 1. 数据库数据源

```json
{
  "name": "sales_data",
  "type": "database",
  "database": "cfs-report",
  "sql": "SELECT region, product, amount, quantity FROM sales WHERE date >= '${startDate}' AND date <= '${endDate}' AND org_id = '${orgId}'"
}
```

**说明：**
- `name`: 数据源名称（自定义）
- `type`: 固定为 `database`
- `database`: 数据库连接名称
- `sql`: SQL 查询语句，使用 `${参数名}` 作为占位符

**SQL 参数示例：**
```sql
-- 参数会从筛选组件获取值
SELECT * FROM sales 
WHERE date >= '${startDate}' 
  AND date <= '${endDate}'
  AND org_id = '${orgId}'
```

---

### 2. Class 数据源

```json
{
  "name": "credit_data",
  "type": "class",
  "class_name": "com.yocyl.fr.engine.tableData.finance.CreditContractDetailData",
  "return_fields": [
    {"name": "contractNo", "type": "string"},
    {"name": "amount", "type": "number"}
  ],
  "parameter_template": {
    "orgId": "",
    "startDate": "2026-03-01",
    "endDate": "",
    "indexInfo": []
  }
}
```

**说明：**
- `name`: 数据源名称（自定义）
- `type`: 固定为 `class`
- `class_name`: Java 类全路径
- `return_fields`: 返回字段列表
  - `name`: 字段名
  - `type`: 字段类型（string/number/date）
- `parameter_template`: 入参模板
  - **空值 `""`** → 从筛选组件获取
  - **有值 `"2026-03-01"`** → 写死的默认值
  - **空数组 `[]`** → 从筛选组件获取（数组类型）

**入参模板规则：**
```json
{
  "orgId": "",              // 空 → 需要配置筛选组件 orgId
  "startDate": "2026-03-01", // 有值 → 固定默认值
  "indexInfo": []           // 空数组 → 需要配置筛选组件 indexInfo
}
```

## 列映射配置

将 Excel 列（A, B, C...）映射到数据库字段或 Class 返回字段：

```json
{
  "A": "region",
  "B": "product",
  "C": "amount",
  "D": "quantity"
}
```

**说明：**
- Key: Excel 列号（A=第一列, B=第二列...）
- Value: 对应的字段名
- **不支持一对多**：一个 Excel 列只能映射一个字段

**示例：**
```
Excel:     A列      B列      C列      D列
           ↓        ↓        ↓        ↓
映射:    region  product  amount  quantity

生成的报表：
| 区域 | 产品 | 金额 | 数量 |
|------|------|------|------|
| 华东 | A    | 1000 | 50   |
```

## 筛选组件配置

### 组件结构（5 对组件一行）

```
开始日期 —— [日期控件]   组织机构 —— [树形下拉]   区域 —— [下拉框]
   ↑          ↑             ↑          ↑           ↑        ↑
 文本组件    传值组件      文本组件   传值组件    文本组件  传值组件
```

### JSON 配置

```json
[
  {
    "label": "开始日期",
    "code": "startDate",
    "type": "DateEditor",
    "default_value": ""
  },
  {
    "label": "组织机构",
    "code": "orgId",
    "type": "TreeComboBoxEditor",
    "default_value": ""
  },
  {
    "label": "合同编号",
    "code": "contractNo",
    "type": "TextEditor",
    "default_value": "",
    "placeholder": "请输入合同号模糊查询"
  },
  {
    "label": "区域",
    "code": "region",
    "type": "ComboBox",
    "default_value": "",
    "options": {
      "east": "华东",
      "south": "华南",
      "north": "华北"
    }
  }
]
```

**字段说明：**
| 字段 | 说明 | 示例 |
|------|------|------|
| `label` | 文本组件（说明），用户看到的标签 | "开始日期" |
| `code` | 参数名（传值组件），用于数据集参数绑定 | "startDate" |
| `type` | 控件类型 | DateEditor |
| `default_value` | 默认值 | "" |
| `placeholder` | 占位提示 | "请选择..." |
| `options` | 下拉选项（仅 ComboBox） | {"key": "显示值"} |

### 支持的控件类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| `TextEditor` | 文本输入框 | 合同号、名称等 |
| `DateEditor` | 日期选择器 | 开始日期、结束日期 |
| `ComboBox` | 下拉选择框 | 区域、状态等固定选项 |
| `TreeComboBoxEditor` | 树形下拉 | 组织机构、部门等层级结构 |
| `NumberEditor` | 数字输入框 | 金额、数量等 |
| `ComboCheckBox` | 多选下拉 | 多个选项 |

## 参数绑定关系

```
┌─────────────────────────────────────────────────────────────┐
│                    参数绑定流程                              │
└─────────────────────────────────────────────────────────────┘

筛选组件配置:
┌───────────────────────────────────────────────────────────┐
│ 开始日期(startDate)  组织机构(orgId)  区域(region)         │
└───────────────────────────────────────────────────────────┘
          │                  │               │
          ▼                  ▼               ▼
数据源入参匹配:
┌───────────────────────────────────────────────────────────┐
│ 数据库 SQL: ...WHERE date >= '${startDate}'                │
│            AND org_id = '${orgId}'                         │
│                                                            │
│ Class 入参: {"orgId": "", "startDate": "", "region": ""}   │
└───────────────────────────────────────────────────────────┘

✅ 自动匹配: startDate, orgId, region 都有对应筛选组件
⚠️ 警告: indexInfo 无对应组件 → 需要添加或设置默认值
```

## 常见问题

### Q1: JSON 格式错误

**错误**: `Expected ':' after property name in JSON`

**原因**: JSON 格式不正确，常见问题：
- 属性名没有用双引号
- 使用了单引号而不是双引号
- 最后一个元素后面多了逗号

**正确示例**:
```json
{"name": "field1", "type": "string"}
```

**错误示例**:
```json
{name: "field1", 'type': 'string'}  // ❌ 缺少引号或用了单引号
{"name": "field1",}  // ❌ 最后多了逗号
```

### Q2: 参数绑定警告

**现象**: 提示"xxx 参数无对应筛选组件"

**解决**:
1. 检查数据源入参中的参数名
2. 添加对应的筛选组件，确保 `code` 与参数名一致
3. 或者在入参模板中设置默认值

### Q3: 列映射不生效

**检查**:
1. Excel 列号是否正确（A=第一列）
2. 字段名是否与数据源返回字段/SQL 字段一致
3. 不要使用小写列号（用 A 不是 a）

## 快速复制模板

### 数据库数据源模板
```json
{
  "name": "数据源名称",
  "type": "database",
  "database": "数据库名",
  "sql": "SELECT 字段1, 字段2 FROM 表名 WHERE 条件 = '${参数名}'"
}
```

### Class 数据源模板
```json
{
  "name": "数据源名称",
  "type": "class",
  "class_name": "com.xxx.XXXData",
  "return_fields": [
    {"name": "字段1"},
    {"name": "字段2"}
  ],
  "parameter_template": {
    "参数1": "",
    "参数2": "默认值"
  }
}
```
